EBSD_300 work log migrated from monolithic log_book artifact format to the new individual entries format.

Migration was driven by a observed performance slowdown on very large log books — the monolithic format required reading and rewriting the entire log file on every add_log_entry call, causing noticeable latency as the log grew. The new format stores each entry as a separate entries/*.md file indexed by index.json, so appending a new entry touches only two small files regardless of total log size. Token cost per operation scales with the entry, not the cumulative log history.

The render_log_book tool assembles the full log on demand for reading. Individual entry reads remain lightweight. This change is transparent to the engineer — the log content and entry types are unchanged.
